# ch03.sec07 상속은 언제 사용할까??

    코드의 재사용에 있어서, 객체 지향 프로그래밍에서는 크게 2가지 방법을 사용할 수 있다.
     1. 상속(Inheritance)
     2. 구성(Composition) <- 일반적인 코드 재사용 방법

## IS-A 관계
1. 개념 관계 : 일반적인(general) 개념 - 구체적인(specific) 개념
    - 상위 클래스 : 하위 클래스보다 일반적인 개념(ex. Employee)
    - 하위 클래스 : 상위 클래스보다 구체적인 개념(ex. Engineer, Manager...)

2. 상속한다.
    - 클래스 간의 결합도가 높은 설계
    - 상위 클래스의 수정 -> 많은 하위 클래스에 영향을 미칠 수 있다.
    - 계층 구조가 복잡하거나 hierachy가 높을 경우 NG
 
## HAS-A 관계

1. 개념 관계 : 클래스가 다른 클래스를 포함하는 관계(클래스를 변수로 선언)
    - 코드 재사용의 가장 일반적인 방법
    - Library 구현 시, ArrayList를 생성하여 사용한다.

2.  상속하지 않는다.
