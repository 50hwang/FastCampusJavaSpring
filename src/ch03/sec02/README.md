# ch03.sec02 상속을 활용한 멤버십 클래스 구현하기

    회사에서 고객 정보를 활용한 서비스를 하기 위해, 일반고객(Customer)와 이보다 충성도가 높은 우수고객(VIPCustomer)에 따른 서비스를 제공하고자 함.
    물품을 구매할 때 적용되는 할인율과 적립되는 보너스의 포인트 비율이 다름.
    여러 멤버십에 대한 각각 다양한 서비스를 제공할 수 있음.
    이 따, 멤버십에 대한 구현을, _클래스 상속_ 을 활용하여 구현해보자.


## 상위 클래스 : 일반 고객(Customer) 클래스 구현

- 고객의 속성
    - 고객 아이디
    - 고객 이름
    - 고객 등급
    - 보너스 포인트
    - 보너스 포인트 적립 비율
 
- 예제 코드 : Customer.java
 
## 하위 클래스 : 우수 고객(VIPCustomer) 클래스 구현

- 우수 고객에게 추가 사항
    - (다르게 적용된) 보너스 포인트 적립 비율
    - (다르게 적용된) 결제 금액에 따른 고객 등급 차별화
    - 할인율 적용
    - 전담 상담원 배정
 
- Customer 클래스에 추가해서 구현하는 것은 NG
- 심지어, 수시로 코드 세부 속성 변경을 요구할 수도 있음
    - ex) "일반 고객과 우수 고객 사이에 준우수 고객 추가하자" 라든가...
- VIPCustomer 클래스를 따로 구현
- 이미 Customer에 구현된 내용 중복 -> Customer를 확장하여 구현
    - 이 때 상속 개념이 위력을 발휘한다!!
    - `class VIPCustomer extends Customer {...}`
    - 상위 클래스의 private 속성을 상속받고 싶다면, 접근 제어자를 `private` -> `protected`로 바꾸어주어야 한다!!
 
- 예제 코드 : VIPCustomer.java

## protected 접근 제어자
- 상위 클래스에서 `private` 선언을 할 시,
    - 외부 클래스도 X, 상속받는 하위 클래스도 X
 
- 외부 클래스는 접근 X이되, 하위 클래스에서는 O이 되기 위해서는,
     - `protected` 를 사용한다.
