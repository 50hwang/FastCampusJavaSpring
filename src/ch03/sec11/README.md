# ch03.sec11 인터페이스(interface)

    구현 메소드 코드 및 변수 없이, 추상 메소드와 상수만으로 선언된 클래스.

## 인터페이스란?

- 모든 변수가 `public static final` 상수로만 선언됨.
- 모든 메소드가 `public abstract` 추상 메소드로만 선언됨.
 
> `Java 8`부터는 인터페이스에도 디폴트 메소드와 정적 메소드의 일부 구현 코드가 존재하기도 한다.
 
## 인터페이스 정의와 구현
 
> 예제 코드 : 인터페이스 `Calc` <- 추상 클래스 `Calculator` <- 구상 클래스 `CompleteCalc`, 테스트 코드 `CalculatorTest`
 
## 인터페이스 구현과 형 변환

- 인터페이스를 구현한 클래스는 인터페이스 타입으로 선언한 변수로 형 변환할 수 있다.
    - `Calc calc = new CompleteCalc();`
    
- 상속에서의 업 캐스팅과 동일한 의미를 갖고 있다.
    - 즉, 상위 인터페이스에 선언된 메서드만 접근, 사용할 수 있다.
    - 클래스 상속과는 다르게 구현 코드가 없기 때문에, 여러 인터페이스를 구현하여도 모호성이 발생되지 않는다.
    
> 예제 코드의 테스트 코드 `CalculatorTest`에서, `calc.showInfo()` 메소드는 Calc 타입의 객체 변수에서 사용이 불가하다.
