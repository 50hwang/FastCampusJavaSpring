# ch03.sec09 추상 클래스(abstract class) 구현하기

    추상 메소드는 상속에서 메소드 오버라이딩을 위한 모체 역할만,
    추상 클래스는 하위 클래스 상속을 위한 상위 클래스 역할만 가능하며,
    둘 다 자신을 구체화(메소드 정의, 객체 생성)할 수 없다.

## 추상 클래스란?

    추상(abstract) <-> 구상(concrete)

- 추상 메소드(abstract method)를 포함한 클래스
    - 추상 메소드 : 메소드 구현 코드 없이 선언만 존재하는 메소드
       - 선언 : 반환 타입 + 메소드명 + 파라미터
         - 추상 메소드 선언 시, `abstract`라는 예약어를 사용한다.
         - 선언 문장에서 세미콜론(;)으로 끝낸다.
       - 정의 : 메소드 구현 코드 -> 중괄호 {}로 된 바디(body)
         - 추상 메소드는 이 바디 즉, 정의부가 존재하지 않는다.
 
- 추상 클래스는 객체를 생성할 수 없다.
 
## 추상 클래스 구현하기
 
    상속받을 하위 클래스들마다 구현할 재정의 메소드를 상위 클래스에서 모를 경우, 실제 구현 코드를 정의하는 역할을 하위 클래스에 위임할 때 사용한다.
 
    오버라이딩 : 추상은 필수, 구상은 선택
 
- 추상 메소드가 하나라도 존재하면, 무조건 추상 클래스로 정의해야 한다.
    - `abstract`로 선언된 메소드를 가진 클래스를 `abstract`로 선언하지 않으면 오류가 발생한다.

- 추상 메소드가 하나도 없는 구상 클래스도 추상 클래스로 정의할 수는 있다.
    - 대신, 여느 추상 클래스와 마찬가지로, 자체 객체 생성은 할 수 없다.
    - 추상 클래스 안의 구상 메소드 : 하위 클래스가 공통으로 사용하는 메소드.
      - 필요에 따라 구상 메소드도 오버라이딩 할 수도 있다.
 
> 예제 코드 : 
> 1. 상위 추상 클래스 `Computer` <- 하위 구상 클래스 `Desktop`, 하위 추상 클래스 `Notebook`(<- 하위 구상 클래스 `MyNoteBook`)
> 2. 테스트 코드 `ComputerTest`
