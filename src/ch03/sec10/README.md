# ch03.sec10 추상 클래스의 응용 - 템플릿 메서드 패턴
    
    추상 메소드 or 구상 메소드를 활용하여 코드 시나리오를 final로 정의하는, 프레임워크에서 많이 사용되는 메소드 설계 패턴

## 템플릿 메소드

- 추상 클래스로 선언된 상위 클래스에서 템플릿 메소드를 활용하여 전체적인 흐름을 정의
    - `final`로 선언하여, 하위 클래스에서 재정의 방지
 
- 하위 클래스에서 다르게 구현되어야 하는 부분은 하위 클래스에서 재정의 후 구현
 
- 후크 메소드 : 추상 상위 클래스에 구현부가 비어있는 구상 메소드로 정의한 것 -> 특정 하위 클래스의 필요에 따라서만 옵셔널하게 추가하는 것처럼 재정의할 수 있다.

> cf. 라이브러리 vs 프레임워크
> 
>   둘 다 이미 만들어진 무언가를 가져다 쓰긴 하는데,
> - 라이브러리 : 전반적인 프로그램과 코드의 흐름을 개발자가 전부 구성하여 작성
>     - ex) `JDK`
> - 프레임워크 : 전반적인 프로그램과 코드의 흐름이 이미 정해져있어, 구멍(place hole)을 메꾸는 방식
>     - ex) `Spring`, `Android`
