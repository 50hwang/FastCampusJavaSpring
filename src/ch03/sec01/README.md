```
Q. 객체의 "상속"을 쓰는 건, 코드의 "재사용"을 위해서다??

A. 기존 클래스의 기능, 속성 등을 다른 클래서에서 물려받아, 더 나아가 확장하여 사용하기 위해서다.
```

## 클래스 상속

### 상속

- 새로운 클래스를 정의할 때, 이미 구현된 클래스를 물려받아(상속받아) 속성이나 기능을 확장한 하위 클래스로 구현함
 
- 상속의 화살표 방향 : 

    - 하위 클래스 -> 상위 클래스
 
    '하위 클래스가 상위 클래스를 바라본다.'

- 상속의 문법

    - 상위 클래스 A를 상속받아, 하위 클래스 B를 생성할 때,
    - `class B extends A {...}`를 사용한다.
        - Single Inheritance : 기본적으로 상속은 단 하나의 클래스로부터밖에 받을 수 없다.
        - 자바가 추구하던 안정성과 심플함에 따라, 모호성 방지를 위한 제한사항

### 상속을 구현하는 경우

> 기존의 일반적인 코드를 받아다가, 같은 결을 갖지만 보다 더 구체적이고 확장된 개념과 기능을 구현할 때
 
     상위 클래스 : 더 일반적, 포괄적인 개념과 기능
        ^
        | <- (타겟을) 가져다가 속성과 기능을 확장(extends)한다는 의미
        |
     하위 클래스 : 더 구체적, 확장적인 개념과 기능

- 예제 코드

```
class Mammal {...}	//<-상위 클래스(포유류)

class Human extends Mammal {...}	//<-하위 클래스(포유류 개중에서 인간)

```
